Shared framework mod for the Pipes and Power Expanded and Steelmaking Expanded family. It ships no gameplay content of its own - install it only because another mod depends on it. It bundles the systems both of those mods are built on, so they stay in sync and share a single block-network manager rather than one each.

Compatibility

  • Game version: built for Vintage Story 1.22. Legacy 1.20 and 1.21 builds are on the GitHub releases page.
  • Multiplayer: works in singleplayer and on dedicated servers.
  • Requires: nothing - this is itself a dependency for other mods.
  • Languages: English, Russian, Ukrainian.
  • Save safety: safe to add to an existing world. Only remove it once no installed mod depends on it.

What it provides

  • Block networks - a generic connected-graph framework with self-orienting node blocks, merge-on-join and fracture-on-break handling, and a single network manager. ppex registers its pipe network on it and smex its molten network.
  • Multiblock structures - structure-completion monitoring, the build-outline projection (Ctrl + Shift + right-click), crash-safe highlighting of incomplete parts, and the invisible filler block that gives single-cell mega-block machines proper per-cell collision.
  • Block-entity healing - recreates a block entity that was lost while its block survived (a load error or a server desync), so a machine never ends up an inert, unbreakable ghost. Runs on chunk load and on demand via /exmod heal.
  • Live config - every gameplay value is exposed through a JSON config that can also be read and changed in-game with /exmod config, applied immediately with no world reload.
  • Recipe-cost levels - lets each dependent mod offer cheaper or normal crafting and construction costs, switchable with /exmod recipes.
  • Network highlighting - .exmod network hi tints every networked block, a separate colour per network, so you can see which blocks share a run and where one is broken.
  • Save migrations - rewrites renamed block codes (and matching item stacks) in old saves as chunks load, so updates do not leave unknown blocks behind.
  • Shared helpers - orientation and rotation math, particle and sound catalogues, a production-machine base, content gating, and the attribute-driven registration both mods use.

Commands

  • .exmod network hi / unhi - toggle the per-network highlight (client).
  • /exmod config <mod> - view or change a dependent mod's config values live (admin).
  • /exmod recipes <mod> - switch a dependent mod's recipe-cost level (admin).
  • /exmod heal - recreate any orphaned block entities in loaded chunks (admin).

Dependent mods add their own sub-commands under the same /exmod and .exmod root.

FAQ

  • Do I need to configure anything? No. It works out of the box; the dependent mods drive it.
  • Can I remove it? Only once nothing else needs it. Removing it while ppex or smex is installed will break those mods.
  • I see unknown blocks after updating a dependent mod. Migrations run on load; if any slipped through, replace them in creative.
